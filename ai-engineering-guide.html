01 — Communication Layer

Prompt Engineering

Prompting is the primary interface between your intent and an LLM's output. Poor prompts produce inconsistent, hallucinated, and off-format results. Mastery here multiplies every other AI skill you have.

🎯

Be Explicit

State role, task, format, and constraints in every system prompt. Ambiguity compounds across retries.

🔢

Chain of Thought

Ask the model to reason step-by-step before answering. Dramatically improves accuracy on multi-step problems.

📎

Few-Shot Examples

2–5 input/output examples in the prompt steer behaviour more reliably than elaborate instructions alone.

🔲

Output Structuring

Request JSON, XML, or markdown with a schema. Structured output is parseable, testable, and pipe-able.

The Anatomy of a Production System Prompt

System Prompt — Code Review Agent
You are a senior JavaScript/TypeScript engineer performing a security-focused code review. ## Your Role - Identify bugs, security vulnerabilities, performance issues, and style violations - Prioritise findings by severity: CRITICAL > HIGH > MEDIUM > LOW - Explain *why* each issue matters, not just that it exists - Suggest concrete fixes with code examples ## Output Format Respond ONLY with valid JSON matching this schema exactly: { "summary": "string — one sentence overview", "score": number (0-10, where 10 is production-ready), "findings": [ { "severity": "CRITICAL|HIGH|MEDIUM|LOW", "category": "security|performance|correctness|style", "line": number | null, "issue": "string", "fix": "string", "codeExample": "string | null" } ] } ## Rules - Never hallucinate line numbers. If unsure, set line to null. - If the code is safe and correct, return an empty findings array with score 9 or 10. - Do not add commentary outside the JSON object.

// ── 2. Self-consistency — sample, then vote ────────
async function selfConsistency(prompt, n = 5) {
  const answers = await Promise.all(
    Array.from({ length: n }, () => llm(prompt, { temperature: 0.7 }))
  )
  // Majority vote on structured answers
  const freq = Map.groupBy(answers, a => a.verdict)
  return [...freq].sort((a, b) => b[1].length - a[1].length)[0][0]
}

// ── 3. Prompt chaining — decompose complex tasks ──
async function reviewAndFix(code) {
  // Step 1: identify issues
  const review = await llm(reviewPrompt(code))

  // Step 2: fix each CRITICAL/HIGH issue (parallel)
  const criticals = review.findings.filter(f => f.severity === 'CRITICAL')
  const fixes = await Promise.all(
    criticals.map(f => llm(fixPrompt(code, f)))
  )

  // Step 3: merge fixes into final output
  return llm(mergePrompt(code, fixes))
}

02 — Information Architecture

Context Engineering

Context engineering is the discipline of deciding what information goes into the context window, in what form, and in what order — so the model has exactly what it needs and nothing that dilutes or contradicts it.

As context windows grew from 4k to 1M+ tokens, the bottleneck shifted from what can fit to what should be included. More is not always better. Irrelevant context increases cost, degrades retrieval quality, and can confuse the model.

"Prompt engineering is what you say. Context engineering is what you include and how you structure it."

The Context Window Hierarchy

JavaScript — Context Builder Pattern
// ── Structured context assembly ───────────────────
// Organise context into layers — high signal first
class ContextBuilder {
  #sections = []
  #tokenBudget

  constructor(maxTokens = 32_000) {
    this.#tokenBudget = maxTokens
  }

  // Add in priority order — system > instructions > retrieved > history > query
  addSystemInstructions(text) {
    this.#sections.push({ priority: 0, label: 'SYSTEM', text })
    return this
  }
  addRetrievedDocs(docs) {
    const text = docs
      .map((d, i) => `[DOC ${i+1} | score:${d.score.toFixed(2)}]\n${d.content}`)
      .join('\n\n')
    this.#sections.push({ priority: 1, label: 'RETRIEVED CONTEXT', text })
    return this
  }
  addConversationHistory(turns, maxTurns = 6) {
    // Keep only recent turns — older context dilutes signal
    const recent = turns.slice(-maxTurns)
    const text = recent.map(t => `${t.role.toUpperCase()}: ${t.content}`).join('\n')
    this.#sections.push({ priority: 2, label: 'CONVERSATION HISTORY', text })
    return this
  }
  addUserQuery(query) {
    this.#sections.push({ priority: 3, label: 'USER QUERY', text: query })
    return this
  }

  build() {
    return this.#sections
      .sort((a, b) => a.priority - b.priority)
      .map(s => `<${s.label}>\n${s.text}\n</${s.label}>`)
      .join('\n\n')
  }
}

// ── Conversation summarisation — avoid token bloat ─
async function summariseHistory(messages) {
  if (estimateTokens(messages) < 4_000) return messages

  const summary = await llm([{
    role:    'user',
    content: `Summarise this conversation in 3-5 bullet points.
              Preserve all technical decisions and constraints:\n\n${messages}`
  }])

  // Replace old turns with compact summary
  return [
    { role: 'system', content: `Previous conversation summary:\n${summary}` },
    ...messages.slice(-2)  // keep only last exchange verbatim
  ]
}

Token Counting & Budget Management

JavaScript — Token Estimation & Budgeting
import { encode } from 'gpt-tokenizer'  // works for GPT-4, Claude approx.

// ── Estimate token count before sending ──────────
function estimateTokens(text) {
  if (typeof text === 'string') return encode(text).length
  if (Array.isArray(text))  // message array
    return text.reduce((n, m) => n + encode(m.content).length + 4, 3)
}

// ── Context window budget allocation ─────────────
//  Model limit:     128,000 tokens
//  System prompt:    ~2,000
//  Retrieved docs:  ~20,000
//  History:          ~4,000
//  User query:       ~1,000
//  Reserve for reply:~8,000
//  ─────────────────────────
//  Total used:      ~35,000  ✓ well within limit

const BUDGET = {
  system:    2_000,
  docs:      20_000,
  history:   4_000,
  query:     1_000,
  reply:     8_000,
}

// Truncate docs to fit within budget
function fitDocsToBudget(docs, budget) {
  let used = 0
  return docs.filter(doc => {
    const tokens = estimateTokens(doc.content)
    if (used + tokens > budget) return false
    used += tokens
    return true
  })
}
🧠 Lost in the Middle
Research consistently shows LLMs perform worst on information placed in the middle of long contexts. Put the most important instructions at the beginning (system prompt) and repeat key constraints at the end (just before the user query). Critical information buried in a 100k-token middle is often ignored.

AI Code Reviews

AI code review augments human review — it never replaces it. AI excels at catching categories of issues consistently and at scale: security anti-patterns, performance smells, missing error handling, and style drift.

JavaScript — Automated AI Review in CI
import Anthropic from '@anthropic-ai/sdk'
import { execSync } from 'node:child_process'

const client = new Anthropic()

async function reviewPullRequest(baseBranch = 'main') {
  // Get only the changed files diff
  const diff = execSync(
    `git diff ${baseBranch}...HEAD -- '*.ts' '*.js' '*.tsx'`
  ).toString()

  if (!diff.trim()) { console.log('No JS/TS changes'); return }

  const response = await client.messages.create({
    model:      'claude-opus-4-5',
    max_tokens: 4096,
    system: `You are a senior engineer reviewing a pull request diff.
Focus on: security vulnerabilities, unhandled errors, N+1 queries,
missing input validation, race conditions, and resource leaks.
Respond in JSON: { score: 0-10, blockers: [], warnings: [], suggestions: [] }
Each item: { severity, file, description, fix }`,
    messages: [{
      role:    'user',
      content: `Review this diff:\n\`\`\`diff\n${diff.slice(0, 60_000)}\n\`\`\``
    }]
  })

  const review = JSON.parse(response.content[0].text)

  // Fail CI if blockers exist
  if (review.blockers.length > 0) {
    console.error('🚨 AI Review blockers:', review.blockers)
    process.exit(1)
  }
  return review
}

05 — Retrieval Systems

RAG Fundamentals

Retrieval-Augmented Generation (RAG) grounds LLM responses in your data — documentation, codebases, knowledge bases — without the cost or latency of fine-tuning. It is the most practical pattern for production AI applications.

✂️

Chunking

Split documents into semantically coherent chunks (500–1000 tokens). Overlap by ~20% to preserve context at boundaries.

🔢

Embedding

Convert chunks to dense vectors using an embedding model (text-embedding-3-small, Cohere, or nomic-embed). Store in a vector DB.

🔍

Retrieval

At query time, embed the query and find the top-k nearest chunks by cosine similarity. Hybrid search (vector + BM25) beats either alone.

🤖

Generation

Feed retrieved chunks + query to the LLM. Instruct it to answer only from provided context. Cite sources in the output.

JavaScript — RAG Pipeline with pgvector
import OpenAI from 'openai'
import { prisma } from './db.js'

const openai = new OpenAI()

// ── Step 1: Embed and store documents ────────────
async function ingestDocument(doc) {
  const chunks = chunkText(doc.content, { size: 800, overlap: 160 })

  // Batch embed for efficiency (up to 2048 inputs at once)
  const embedRes = await openai.embeddings.create({
    model: 'text-embedding-3-small',
    input: chunks.map(c => c.text),
  })

  await prisma.$executeRaw`
    INSERT INTO document_chunks (doc_id, chunk_index, content, embedding, metadata)
    SELECT * FROM UNNEST(
      ${chunks.map(c => doc.id)}::uuid[],
      ${chunks.map((_, i) => i)}::int[],
      ${chunks.map(c => c.text)}::text[],
      ${embedRes.data.map(e => JSON.stringify(e.embedding))}::vector[],
      ${chunks.map(c => JSON.stringify(c.metadata))}::jsonb[]
    )
  `
}

// ── Step 2: Hybrid retrieval (vector + keyword) ──
async function retrieve(query, topK = 5) {
  const qEmbed = await openai.embeddings.create({
    model: 'text-embedding-3-small', input: query
  })
  const qVec = JSON.stringify(qEmbed.data[0].embedding)

  // Reciprocal Rank Fusion of vector + BM25 results
  return prisma.$queryRaw`
    WITH vector_results AS (
      SELECT id, content, metadata,
             1 - (embedding <=> ${qVec}::vector) AS score
      FROM document_chunks
      ORDER BY embedding <=> ${qVec}::vector
      LIMIT ${topK * 2}
    ), keyword_results AS (
      SELECT id, content, metadata,
             ts_rank(to_tsvector('english', content),
                     plainto_tsquery('english', ${query})) AS score
      FROM document_chunks
      WHERE to_tsvector('english', content) @@
            plainto_tsquery('english', ${query})
      LIMIT ${topK * 2}
    )
    SELECT id, content, metadata,
           (COALESCE(v.score,0) * 0.7 + COALESCE(k.score,0) * 0.3) AS combined_score
    FROM vector_results v FULL JOIN keyword_results k USING (id)
    ORDER BY combined_score DESC LIMIT ${topK}
  `
}

// ── Step 3: Generate grounded answer ─────────────
async function ragAnswer(question) {
  const docs = await retrieve(question)
  const context = docs
    .map((d, i) => `[${i+1}] ${d.content}`)
    .join('\n\n')

  const res = await openai.chat.completions.create({
    model:    'gpt-4o-mini',
    messages: [
      { role: 'system', content:
        'Answer using ONLY the provided context. If the answer is not in the context, say so. Cite source numbers like [1].' },
      { role: 'user',    content:
        `Context:\n${context}\n\nQuestion: ${question}` }
    ]
  })
  return { answer: res.choices[0].message.content, sources: docs }
}
💡 RAG Failure Modes
The top three RAG failures: bad chunking (splits mid-sentence, losing context), wrong retrieval (top-k returns irrelevant docs — add a relevance threshold, discard <0.75), and faithfulness drift (model ignores context and answers from training data — add explicit grounding instructions and use faithfulness evals).
06 — Autonomous Systems

Agents & Tool Calling

An AI agent is an LLM that can take actions in the world — calling APIs, running code, querying databases, browsing the web — in a loop until a goal is achieved. Tool calling is the mechanism that makes this possible.

JavaScript — Tool Calling with Anthropic SDK
import Anthropic from '@anthropic-ai/sdk'

const client = new Anthropic()

// ── Define tools with JSON Schema ─────────────────
const tools = [
  {
    name:        'search_codebase',
    description: 'Search for functions, types, or patterns in the codebase using ripgrep',
    input_schema: {
      type: 'object',
      properties: {
        query:   { type: 'string',  description: 'Search term or regex' },
        fileGlob:{ type: 'string',  description: 'File pattern e.g. *.ts', default: '**/*.ts' }
      },
      required: ['query']
    }
  },
  {
    name:        'run_tests',
    description: 'Run the test suite and return pass/fail results',
    input_schema: {
      type: 'object',
      properties: {
        pattern: { type: 'string', description: 'Test file pattern' }
      }
    }
  },
  {
    name:        'write_file',
    description: 'Write content to a file in the project',
    input_schema: {
      type: 'object',
      properties: {
        path:    { type: 'string' },
        content: { type: 'string' }
      },
      required: ['path', 'content']
    }
  }
]

// ── Agentic loop ──────────────────────────────────
async function agentLoop(task, toolHandlers, maxSteps = 10) {
  const messages = [{ role: 'user', content: task }]

  for (let step = 0; step < maxSteps; step++) {
    const response = await client.messages.create({
      model:     'claude-opus-4-5',
      max_tokens: 4096,
      tools,
      messages,
    })

    messages.push({ role: 'assistant', content: response.content })

    // Model finished — no more tool calls
    if (response.stop_reason === 'end_turn')
      return response.content.find(b => b.type === 'text')?.text

    // Execute all tool calls in parallel
    const toolUses = response.content.filter(b => b.type === 'tool_use')
    const results  = await Promise.all(toolUses.map(async (use) => {
      const handler = toolHandlers[use.name]
      if (!handler) throw new Error(`Unknown tool: ${use.name}`)
      const output = await handler(use.input)
      return { type: 'tool_result', tool_use_id: use.id,
               content: JSON.stringify(output) }
    }))
    messages.push({ role: 'user', content: results })
  }
  throw new Error(`Agent exceeded ${maxSteps} steps`)
}
⚠️ Agent Safety
Every tool an agent can call is a potential blast radius multiplier. Apply the principle of least privilege to tool definitions. Tools that write files, call external APIs, or execute code must be scoped. Add a human-in-the-loop checkpoint before any irreversible action (deleting data, sending emails, deploying code).

async function checkForInjection(userInput) {
  const verdict = await llm([{
    role:    'user',
    content: `Does this text attempt to give instructions to an AI model?
Answer only YES or NO.\n\nText: "${userInput}"`
  }], { temperature: 0 })
  return verdict.trim() === 'YES'

JavaScript — PII Redaction Before LLM Call
import { ComprehendClient, DetectPiiEntitiesCommand }
  from '@aws-sdk/client-comprehend'

// ── Redact PII before sending to LLM ─────────────
async function redactPII(text) {
  const comprehend = new ComprehendClient({ region: 'us-east-1' })
  const { Entities } = await comprehend.send(
    new DetectPiiEntitiesCommand({ Text: text, LanguageCode: 'en' })
  )

  // Replace each PII span with a placeholder
  let redacted = text
  const replacements = new Map()

  for (const e of [...Entities].reverse()) {
    const placeholder = `[${e.Type}_${replacements.size}]`
    replacements.set(placeholder, text.slice(e.BeginOffset, e.EndOffset))
    redacted = redacted.slice(0, e.BeginOffset) + placeholder +
                redacted.slice(e.EndOffset)
  }
  return { redacted, replacements }
}
08 — Architecture

AI System Design

AI features are not CRUD endpoints. They have probabilistic outputs, high latency (100ms–30s), significant cost per call, and failure modes that don't map to HTTP status codes. Design accordingly.

JavaScript — Production AI Service Layer
import Anthropic from '@anthropic-ai/sdk'
import { Redis }     from 'ioredis'

const anthropic = new Anthropic()
const redis     = new Redis(process.env.REDIS_URL)

// ── Semantic caching — avoid repeat LLM calls ─────
async function cachedLLM(prompt, opts = {}) {
  const cacheKey = `llm:${hashPrompt(prompt)}`
  const cached   = await redis.get(cacheKey)
  if (cached) return JSON.parse(cached)

  const result = await anthropic.messages.create({
    model:      opts.model || 'claude-haiku-4-5-20251001',
    max_tokens: opts.maxTokens || 1024,
    messages:   Array.isArray(prompt) ? prompt : [{ role: 'user', content: prompt }]
  })

  // Cache deterministic results (temperature 0) for 1 hour
  if (!opts.temperature)
    await redis.set(cacheKey, JSON.stringify(result), 'EX', 3600)

  return result
}

// ── Fallback cascade — primary → cheaper → error ─
async function withFallback(prompt) {
  const models = [
    'claude-opus-4-5',
    'claude-sonnet-4-6',
    'claude-haiku-4-5-20251001'
  ]
  for (const model of models) {
    try {
      return await cachedLLM(prompt, { model })
    } catch (err) {
      // Retry next model only on 429 / 529 (rate limit / overload)
      if (![429, 529].includes(err.status)) throw err
      logger.warn({ model, err.status }, 'Model unavailable, trying fallback')
    }
  }
  throw new Error('All AI models exhausted')
}

// ── Streaming response to client ──────────────────
app.post('/api/chat', async (req, res) => {
  res.setHeader('Content-Type', 'text/event-stream')
  res.setHeader('Cache-Control', 'no-cache')

  const stream = anthropic.messages.stream({
    model:    'claude-sonnet-4-6',
    max_tokens: 2048,
    messages: req.body.messages,
  })

  for await (const event of stream) {
    if (event.type === 'content_block_delta')
      res.write(`data: ${JSON.stringify({ text: event.delta.text })}\n\n`)
  }
  res.write('data: [DONE]\n\n'); res.end()
})
09 — Economics

Cost Optimisation

LLM APIs bill per token. At scale, naïve usage compounds into significant costs. The highest-leverage optimisations — caching, model routing, prompt compression, and batching — often cut costs 70–90% with minimal quality loss.

💾

Semantic Caching

Cache LLM responses keyed by a hash of the prompt. For deterministic tasks (extraction, classification), cache hit rates of 40–80% are common.

🔀

Model Routing

Route simple tasks to cheap fast models (Haiku, GPT-4o-mini) and complex reasoning to expensive ones. 80% of requests are "simple".

✂️

Prompt Compression

Remove filler words, use abbreviations in few-shot examples, compress retrieved docs. Reducing input 30% cuts cost 30% with near-zero quality loss.

📦

Prompt Caching

Anthropic and OpenAI support prefix caching. Pin your system prompt + static context as a cached prefix — re-use across requests for 90% cost reduction on that prefix.

JavaScript — Model Router & Cost Tracking
// ── Task complexity classifier → model selection ─
const MODEL_TIERS = {
  cheap:    { id: 'claude-haiku-4-5-20251001',  inputCost:  0.25,  outputCost:  1.25  },  // per 1M tokens
  medium:   { id: 'claude-sonnet-4-6',          inputCost:  3.00,  outputCost: 15.00  },
  premium:  { id: 'claude-opus-4-5',           inputCost: 15.00,  outputCost: 75.00  },
}

function selectModel(task) {
  if (task.type === 'classify' || task.type === 'extract')
    return MODEL_TIERS.cheap
  if (task.requiresReasoning || task.contextLength > 10_000)
    return MODEL_TIERS.premium
  return MODEL_TIERS.medium
}

// ── Cost tracking per request ─────────────────────
function trackCost(model, usage) {
  const tier = Object.values(MODEL_TIERS).find(t => t.id === model)
  const cost =
    (usage.input_tokens  / 1_000_000) * tier.inputCost +
    (usage.output_tokens / 1_000_000) * tier.outputCost
  metrics.histogram('ai.cost.usd', cost, { model })
  return cost
}

10 — Quality Assurance

Human Validation & Evaluation

You cannot ship an AI feature without an eval strategy. Evals are to AI systems what tests are to software — the only way to know if your changes made things better or worse.

JavaScript — Eval Framework
// ── LLM-as-judge evaluation ───────────────────────
async function evaluateResponse({ question, context, response, groundTruth }) {
  const verdict = await llm([{
    role:    'user',
    content: `Evaluate this AI response on three dimensions.
Respond ONLY with JSON.

Question: ${question}
Context provided: ${context}
AI Response: ${response}
Ground truth: ${groundTruth}

Score each 1-5:
- faithfulness: Does the response stick to the provided context? (5=fully grounded, 1=hallucinated)
- correctness: Does it match the ground truth? (5=exact, 1=wrong)
- completeness: Does it address all parts of the question? (5=complete, 1=missing key points)

Format: {"faithfulness":N,"correctness":N,"completeness":N,"reasoning":"string"}`
  }], { temperature: 0 })
  return JSON.parse(verdict)
}

// ── Batch eval against test suite ────────────────
async function runEvalSuite(testCases, systemPrompt) {
  const results = await Promise.all(testCases.map(async tc => {
    const response = await llm(tc.question, { system: systemPrompt })
    const scores   = await evaluateResponse({
      question:    tc.question,
      context:     tc.context,
      response,
      groundTruth: tc.expectedAnswer,
    })
    return { ...tc, response, scores }
  }))

  const avg = (key) =>
    results.reduce((s, r) => s + r.scores[key], 0) / results.length

  return {
    totalCases:    results.length,
    avgFaithfulness:  avg('faithfulness'),
    avgCorrectness:   avg('correctness'),
    avgCompleteness:  avg('completeness'),
    failures: results.filter(r => Object.values(r.scores).some(s < 3))
  }
}

🚨 The Vibe Check is Not an Eval
"It looks right to me" is not an evaluation strategy. AI systems degrade silently — a new model version, a changed prompt, or drift in your retrieval corpus can silently drop quality by 20%. Build an automated eval suite that runs on every deploy. Set a quality gate: fail the deploy if avg correctness drops below your baseline.
11 — Daily Practice

Engineering Workflow Integration

The biggest productivity gains from AI tools don't come from occasional use — they come from integrating AI deeply into every stage of the engineering loop, with well-defined checkpoints where human judgement takes over.

  • 1
    Specification — Use AI to challenge your spec before writing code. Prompt: "What edge cases am I missing? What assumptions am I making that could be wrong?" Rubber duck with a model that has read the whole internet.
  • 2
    Architecture — Generate 3 different design options with trade-offs. AI excels at breadth-first exploration of options space. You make the final call with context the model doesn't have.
  • 3
    Implementation — Write the types and tests first. Feed them to the AI as the contract to implement against. This grounds generation and prevents hallucinated APIs.
  • 4
    Review — Run the automated AI review on your own PR before requesting human review. Fix the low-hanging fruit (missing error handling, obvious security issues) so human reviewers focus on logic and architecture.
  • 5
    Documentation — Generate JSDoc, README sections, and ADRs (Architecture Decision Records) from code + context. AI writes the first draft; you edit for accuracy and tone.
  • 6
    Incident response — Paste stack traces + recent logs + recent git diff. Ask AI to generate hypotheses ranked by likelihood. Dramatically accelerates root cause analysis on unfamiliar systems.
